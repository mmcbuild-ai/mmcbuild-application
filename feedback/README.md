# Feedback Files — How This Works

Hi Karen and Karthik,

These files are how you submit design feedback and QA issues for the MMC Build platform. Each of you has your own folder.

---

## Your files

- **Karen:** `feedback/karen/sprint-4-design.md` — design items, UI changes, brand feedback
- **Karthik:** `feedback/karthik/sprint-4-qa.md` — bugs, technical issues, QA findings

---

## How to add an item

1. Go to your file in GitHub (you'll get a direct link from Dennis)
2. Click the **pencil icon** (top-right of the file) to edit
3. Add a new row to the table — fill in your columns, leave "Dennis Decision" as `Pending`
4. Click the green **"Commit changes"** button at the bottom
5. That's it — Dennis gets notified automatically

---

## What happens next

1. Dennis reviews your items and marks the **Dennis Decision** column:
   - **Accept** — will be built in the current sprint
   - **Reject** — won't be built (Dennis will add a note explaining why)
   - **Defer** — good idea, but not this sprint
   - **Pending** — not yet reviewed
2. Only items marked **Accept** get implemented
3. Once built and verified, items move to the "Completed Items" table at the bottom

---

## Tips

- One item per row — keep items specific and actionable
- Include screenshots or links in the Detail column if helpful (paste image URLs or Figma links)
- If something is urgent or blocking, say so in your Status column (`Blocker`)
- You can edit your own files anytime — just don't edit Dennis's Decision column
